# 智能工作流

## 核心概念
- **Workflow**：由触发器、处理器、动作三部分组成。
- **触发器**：CoreBox 命令、剪贴板事件、定时任务、外部 Webhook。
- **处理器**：运行脚本、调用插件 API、执行条件判断。
- **动作**：展示结果、写入存储、发送通知或调用下一 Workflow。

## 快速搭建
1. 打开 Workflow Studio，点击「新建」并选触发器（如“剪贴板含图片”）。
2. 拖入处理模块（OCR、翻译、格式化等），使用连线定义顺序。
3. 设置动作（保存到 Clipbox、推送到 Widget、发送到第三方）。
4. 点击「实时日志」确认每个节点的输入输出，保存即可。

## 模板精选
| 模板 | 说明 |
| --- | --- |
| 智能归档 | 图片 → OCR → 分类 → 文件夹同步 |
| Code Review | Git Hook → AI 总结 → 通知团队 |
| 灵感流水线 | 语音 → 文本 → 标签 → Drafts |

## 调试技巧
- 启用「逐步运行」可定位具体节点输出。
- 使用 `workflow:trace` 命令生成分析报告。
- 通过 Channel API 将调试信息输出到 Terminal Capsule。

## 性能建议
- 将高频 Workflow 部署在本地守护进程，减少唤醒时间。
- 合并相邻的转换节点，使用脚本一次处理。
- 对网络调用启用缓存，提升稳定性。
